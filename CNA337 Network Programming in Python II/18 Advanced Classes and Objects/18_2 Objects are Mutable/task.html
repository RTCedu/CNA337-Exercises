18.2 Objects are Mutable

We can change the state of an object by making an assignment to one of its instance variables. For example, we could change the ip by assigning a new value to self.ip.

We could also have an external function modify self.ip. This type of function is often called a helper function. Second, the request_dhcp_ip method does not return anything. Its job is to modify the object itself. This type of method is known as a mutator method because it mutates or changes the internal state of the object.

Problem

We want to create a DHCP service that assigns a random, unique IP with subnet mask 255.0.0.0. This means the service only sets the last 3 octets of an IP. For example if the dhcp_service function is called by an IP of "192.0.0.0", it should keep the 192 and change the rest:

1. Create a dhcp_service(ip_address) function, outside of the ServerClass

2. Have request_dhcp_ip call the dhcp_service and set the ServerClass object's IP to the generated IP
